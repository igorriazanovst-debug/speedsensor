|-----------|----------|

| Датчик | Оптический датчик прерывания (GND, VCC, D0, A0) |

### Важно
- Обязателен `#include <Adafruit_TinyUSB.h>`
- Датчик логически инвертирован: HIGH = луч свободен, LOW = луч перекрыт
- Прерывание: `FALLING`, `INPUT` (без PULLUP)

## Прошивка (sensor_counter.ino)

Вывод в Serial каждые 500 мс:
```
Pulses: N | RPS: X.XXX | RPM: X.X | V: X.X mm/s | Pin: HIGH/LOW
```
Команды: `t`, `r`, `i`, `h`

Формула: `rps = pulseCount / SLOTS / (elapsed_ms / 1000.0)`

### Поток экранов (QStackedWidget)

```
[Сценарии] → Запустить сценарий
    ↓
[Выбор режима] (mode_selector.py)
    ├── 🔌 Реальный датчик
    │       ↓
    │   [Подключение датчика] (sensor_connect_widget.py)
    │   пульсирующий индикатор, автопоиск каждые 2с, probe по команде 'i'
    │       ↓ успешно → connected(port, baud)
    │   [Эксперимент] — без панели симуляции
    │
    └── 🧪 Моделирование
            ↓ сразу
        [Эксперимент] — с панелью симуляции слева

Кнопка «↩ Сменить режим» → назад к [Выбор режима]
```

### Вкладки главного окна
1. **📋 Сценарии** — список + настройки (единицы, интервал, порт, роль)
2. **📈 Эксперимент** — поток экранов выше
3. **🔬 Обработка** — заглушка (следующий этап)
4. **🌊 Моделирование** — заглушка (следующий этап)

### Вкладка эксперимента
- **График** (pyqtgraph): скользящее окно по X, автомасштаб Y
  - пунктирная линия уставки (только в режиме симуляции)
  - **перекрестие + tooltip при наведении мыши:**
    - прилипает к ближайшей точке кривой
    - показывает: t, ω рад/с, об/с, RPM, V мм/с
    - полупрозрачное скруглённое окошко, следует за курсором
- **Таблица**: Время / ω рад/с / ω об/с / RPM / V мм/с
  - дозаполнение в реальном времени (буфер 100 мс)
  - экспорт CSV (UTF-8-sig, разделитель `;`) и XLSX (стилизованный)

### Режим симуляции (motor_sim.py)
Физическая модель: `J * dω/dt = τ_drive - τ_friction`
Настраивается: скорость (слайдер + точная уставка), инерция, момент привода,
трение, диаметр/масса диска, прорези, уровень шума, джиттер

### Автообнаружение датчика (port_scanner.py)
- Сканирование каждые 2 сек
- Probe: открывает порт 115200, шлёт `i\n`, ждёт ответа с `rps:` / `pulses:`
- Эмитирует: `sensor_found(PortInfo)`, `sensor_lost()`, `ports_updated(list)`

### Парсинг Serial
```python
pattern = re.compile(r"RPS:\s*([\d.]+)")
# rps -> omega = rps * 2 * pi
```
